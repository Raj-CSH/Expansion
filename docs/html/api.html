API¶

expansion module¶

This is the Expansion API, a generative art package with high levels of abstraction, specifically pertaining to a point(s) reproducing in an image, with changing colors, and even environment-sensitive reproduction, with obstacles.

class expansion.ColoredPoint(length, coords, rgb, color_instruction, environment_sensitive)¶

Bases: object

Reproductive point which stores its position and color. All attributes are read-only. Object instances do not define a __dict__, due to the use of __slots__ for decreased size and increased speed.

Parameters

  • length (int) – Side length of square numpy array of which this point belongs to.

  • coords (tuple(int)) – A tuple of integers, consisting of x, y coordinates in the range(0, length).

  • rgb (tuple(float)) – A tuple of 3 floats scaled between 0 and 1 to represent the RGB color value of the point.

  • color_instruction (expansion.colors.ColorInstruction) – Instance of a subclass of expansion.colors.ColorInstruction that defines methods to determine color of reproduced point.

  • environment_sensitive (bool) – Boolean value, defaults to False, if True selects _reproduce_environment_sensitive() function as reproduction method.

reproduce(arr=None)¶

Reproduces point with altered color and position by generating new instances of ColoredPoint. Checks to see whether 4 immediate directions of point are within bounds, before changing color by a small increment and instantiating a new ColoredPoint. If expansion.ColoredPoint.environment_sensitive is enabled then will check if an obstacle is in the way, colored in white(RGB = [1.0, 1.0, 1.0]), before instantiating new ColoredPoints.

Parameters

arr (numpy.ndarray, optional) – An image, in the form of a numpy array with 3 ndimensions, and float values scaled between 0 and 1. Defaults to None, for when environment_sensitive is False, however should be passed an array when environment_sensitive is True.

Returns

A list of the new ColoredPoint

instances that have met the criteria.

run_callbacks(epoch, callbacks=None)¶

Executes callbacks on points given an epoch number.

Parameters

  • epoch (int) – Epoch number to pass to callbacks.

  • callbacks (iterable(expansion.callbacks.Callbacks), optional) – expansion.callbacks.Callback objects to execute. Defaults to no callbacks.

simulate(epochs=0, callbacks=None, close_pool_on_end=True)¶

Executes expansion.ColoredPointHandler.reproduce_points(), expansion.ColoredPointHandler.kill_competitors(), expansion.ColoredPointHandler.render_points(), expansion.ColoredPointHandler.run_callbacks() in that order for a given number of epochs or until image is wholly colored.

Parameters

  • epochs (int) – Number of epochs to simulate, if 0 will simulate until image is wholly colored. Defaults to 0.

  • callbacks (iterable(expansion.callbacks.Callback), optional) – Callbacks to run once expansion.ColoredPointHandler.points have been reproduced, duplicates deleted, and array updated. Defaults to no callbacks.

  • close_pool_on_end (bool) – Whether to close multiprocessing.pool.Pool once function has terminated or leave it running. Defaults to True.

expansion.expansion module¶

This is the main module in the expansion package. Contains the main classes and functions to be used when importing the expansion package. Members can be imported as expansion.name rather than expansion.expansion.name.

expansion.expansion._M = [False, None, 1]¶

These are mutable flags to be passed between script and module. Not meant to be accessed directly, but rather through wrapper functions, such as expansion.enable_multiprocessing() or expansion.is_multiprocessing().

Notes

_M[0] = multiprocessing _M[1] = pool _M[2] = core_count

expansion.utils.stitch(directory, name, fps, dim, f_format, fourcc, frame_ff)¶

Generates a video with a given fourcc Uses frames in a directory generated by the expansion.callbacks.Sample callback used during expansion.ColoredPointHandler.simulate. Frames must be numbered, e.g. ‘1.png’.

Parameters

  • directory (str) – Directory of the frames.

  • name (str) – Desired name of the video.

  • fps (int) – Desired frame rate of the video.

  • dim (tuple(int)) – A tuple consisting of two integers, representing the dimensions of each frame.

  • f_format (str) – Desired file format of the video, e.g. ‘mp4’.

  • fourcc (str) – Desired fourcc of the video, e.g. ‘h264’. You can visit https://www.fourcc.org/ for a detailed list of fourcc codes. Note: you must have the desired fourcc installed on your system to generate a video with that particular fourcc.

  • frame_ff (str) – File format of frames, e.g. ‘png’.
